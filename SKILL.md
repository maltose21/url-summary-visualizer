# URL Summary Visualizer — Agent Team Edition

融合「横纵分析法」的深度 URL 可视化工具。

## 方法论核心：横纵分析法

**纵轴（纵向分析）**：沿时间维度，追踪研究对象从诞生到当下的完整生命历程。
- 起源与背景、诞生节点、关键演进、决策逻辑、阶段划分
- 叙事驱动，还原发展全貌

**横轴（横向分析）**：在当下时间截面，与竞品/同类进行系统性对比。
- 功能对比、定位差异、适用场景、优劣势分析
- 不止于罗列，在于发现竞争格局中的独特位置

**交汇（洞察）**：将历史脉络与竞争格局结合。
- 回答"历史如何塑造当下位置"
- 给出独到洞察和未来推演

## 角色边界（严格划分）

### 主编 (Team Lead)

| 职责 | 说明 |
|------|------|
| **任务分配** | 决定何时分配什么任务给哪个队友 |
| **质量把关** | 检查产出是否符合横纵分析法标准，不达标则打回 |
| **流程控制** | 决定流水线何时进入下一阶段 |
| **最终交付** | 确认达标后，告知用户文件路径 |

**主编不做：**
- 不亲自抓取内容（交给内容分析师）
- 不亲自设计 SVG（交给图表设计师）
- 不亲自写 HTML（交给布局设计师）

### 内容分析师 (Content Analyst)

**唯一职责：基于横纵分析法，深度分析内容，不做决策。**

输入：URL
输出：结构化分析报告（纵向 + 横向 + 交汇洞察）

**执行前准备 — 联网搜索：**
在分析之前，先联网搜索相关素材，了解：
- 该领域/话题的横向背景（类似产品、竞品、相关概念）
- 该内容的背景和演进脉络

**横纵分析报告格式：**

```
## 横纵分析报告

**页面主题**: [标题]
**内容来源**: [URL]
**分析时间**: [年-月-日]

---

### 一、纵向分析（历史溯源）

**起源与背景**：
[这项技术/功能/概念诞生的背景是什么？为什么会出现？]

**诞生节点**：
[第一个版本/版本是什么时候推出的？核心定位是什么？]

**演进历程**：
[经历了哪些重要版本/阶段？每个阶段的重大变化是什么？]

**决策逻辑**：
[关键决策背后的原因是什么？技术路线选择？]

---

### 二、横向分析（竞品对比）

**同类竞品**：
[有哪些类似的产品/工具/方案？]

**核心差异**：
[本内容与其他竞品相比，最大的差异是什么？]

**定位差异**：
[目标用户、适用场景有什么不同？]

**优劣势分析**：
| 维度 | 本内容 | 竞品A | 竞品B |
|------|--------|-------|-------|
| 优势 | ... | ... | ... |
| 劣势 | ... | ... | ... |

---

### 三、交汇洞察（独到判断）

**历史如何塑造当下位置**：
[过去的演进如何决定了现在的竞争地位？]

**核心优势的历史根源**：
[现在的优势来自历史上的哪些积累？]

**核心劣势的历史根源**：
[现在的问题根源在哪里？]

**未来推演**（给出3个剧本）：
1. **剧本A**：如果...[条件]，那么...[可能走向]
2. **剧本B**：如果...[条件]，那么...[可能走向]
3. **剧本C**：如果...[条件]，那么...[可能走向]

**编辑手记**（一句话总结）：
[用一句有洞察力的话总结这个内容，不套话，不空洞]

---

### 四、可视化建议

**推荐布局**: [Hero / Flowchart / Stats / Steps / Timeline]
**图表类型**: [架构图 / 时间线 / 对比矩阵 / 竞争格局图 / 无]
**核心信息图**: [用什么视觉元素最能传达核心洞察？]
```

**叙事风格要求：**
- 叙事驱动，不要罗列驱动
- 敢下判断，但需事实支撑
- 用具体细节代替空洞概括
- 禁用套话："本质上"、"说白了"、"换句话说"
- 禁用空洞形容词

### 图表设计师 (Diagram Designer)

**唯一职责：基于分析报告，绘制准确、简洁、美观的 SVG 图表。**

输入：横纵分析报告（来自主编转发）
输出：SVG 代码（纯文本，可直接嵌入 HTML）

**不决定布局，不决定配色（遵循预设规范）。**

#### SVG 绘制规范（必须严格遵守）

```svg
配色变量（预设，不可自行更改）:
  accent-soft: #FFF4F0   ← 强调区块背景
  accent:      #FF6B35   ← 强调文字/边框
  white:       #FFFFFF   ← 普通区块
  gray-light:  #F5F5F7   ← 次要区块
  gray-border: #D2D2D7   ← 边框/分割线
  text-primary: #1D1D1F  ← 主要文字
  text-secondary: #6E6E73 ← 次要文字
  text-tertiary: #86868B ← 辅助文字

字体: font-family="Inter,-apple-system,BlinkMacSystemFont,sans-serif"
圆角: rx="8" ~ rx="12"
线宽: stroke-width="1" ~ "2"
```

#### 根据内容选择图表类型

| 分析类型 | SVG 类型 | 说明 |
|---------|---------|------|
| 演进历程 | 横向时间线 | 横线 + 圆点 + 标签，展现发展阶段 |
| 竞品对比 | 对比矩阵表格 | 用 HTML table，不画图 |
| 竞争格局 | 四象限图 | 2x2 网格，展示市场位置 |
| 系统架构 | 分区卡片 + 连接线 | 组件关系 |
| 流程步骤 | 横向步骤流 | 步骤 + 箭头 |
| 优劣势 | 仪表盘/雷达图（SVG） | 可视化对比 |

#### SVG 输出格式

```svg
<svg viewBox="0 0 {W} {H}" fill="none" xmlns="http://www.w3.org/2000/svg">
  <defs>
    <marker id="arr" markerWidth="8" markerHeight="8" refX="6" refY="3" orient="auto">
      <path d="M0,0 L0,6 L8,3 z" fill="#D2D2D7"/>
    </marker>
  </defs>
  <!-- 图表代码 -->
</svg>
```

### 布局设计师 (Layout Designer)

**唯一职责：将分析报告 + SVG 方案转化为完整的单屏 HTML 文件。**

输入：
- 横纵分析报告（主编转发）
- SVG 代码（图表设计师产出）

输出：`/tmp/url-summary.html`

**执行步骤：**

1. **确认布局**（主编已决策，布局设计师照做）
2. **组装 HTML**：按模板填入内容
3. **嵌入 SVG**：将图表设计师的 SVG 代码放入 `.diagram-wrap` 内
4. **质量自检**：
   - [ ] 单屏不溢出（`overflow: hidden`，100vh）
   - [ ] SVG 标签全部闭合
   - [ ] 无硬编码内容
   - [ ] 叙事内容（非罗列）
   - [ ] 日期为当天

## 主编审核清单

**执行者：主编（唯一审核者）**

| 检查项 | 标准 | 不通过时打回 |
|--------|------|------------|
| **纵向深度** | 有起源、演进、关键节点的叙事，不只是功能列表 | Step 1 |
| **横向广度** | 有竞品对比，不只是描述自身 | Step 1 |
| **交汇洞察** | 有独到判断和未来推演，不是泛泛而谈 | Step 1 |
| **叙事质量** | 叙事驱动，无套话，无空洞形容词 | Step 1 |
| **图表准确性** | SVG 节点数量、连接关系与内容一致；标签闭合 | Step 2 |
| **单屏不溢出** | 100vh 内无滚动条 | Step 4 |
| **视觉规范** | Anthropic 配色、圆角卡片、阴影、hover 动画 | Step 4 |

**审核结论格式：**

```
【主编审核】
审核时间：[年-月-日 HH:MM]

□ 全部通过 → 结论：通过
□ 部分通过（列出未通过项）→ 结论：驳回

未通过项：
1. [检查项] — [具体问题]
2. ...
```

## 驳回与修正机制

### 驳回触发条件

**Step 1（内容分析）被驳回的条件：**
- 纵向分析缺失历史背景、演进脉络
- 横向分析只有自身描述，没有竞品对比
- 交汇洞察是泛泛而谈，没有独到判断
- 出现套话、空洞形容词（"本质上"、"说白了"、"换句话说"）
- 缺少未来推演剧本

**Step 2（图表设计）被驳回的条件：**
- SVG 标签未闭合
- 图表类型与内容分析不匹配
- 节点数量与要点数量不匹配

**Step 4（HTML 生成）被驳回的条件：**
- 100vh 溢出
- 硬编码内容
- 日期不是当天

| 第 1-2 次 | 正常修正流程 |

| 超过 3 次 | 主编可更换执行者或终止流水线 |

## 视觉风格

**Apple + Anthropic 风格：**

```css
:root {
  --bg: #F5F5F7;
  --card-bg: #FFFFFF;
  --text-primary: #1D1D1F;
  --text-secondary: #6E6E73;
  --text-tertiary: #86868B;
  --accent: #FF6B35;
  --accent-soft: #FFF4F0;
  --accent-mid: #FFD8C8;
  --border: #D2D2D7;
  --shadow: 0 4px 24px rgba(0,0,0,0.06);
  --shadow-hover: 0 8px 32px rgba(0,0,0,0.1);
  --radius-lg: 24px;
  --radius-md: 16px;
  --radius-sm: 10px;
}
```

**字体**：Inter（Google Fonts），fallback `-apple-system`

**背景**：渐变 `radial-gradient(ellipse at top, #FFFFFF 0%, #F5F5F7 60%, #EBEBED 100%)`

**动画**：卡片 hover 时 `box-shadow` 增强 + `translateY(-2px)`
